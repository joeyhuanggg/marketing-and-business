# Report Modes

This file defines the default reporting modes for the 欧美澳商务 BI skill.

## Half-Week

- meeting cadence: Friday
- MTD scope: month start to Thursday
- change window: Tuesday to Thursday
- purpose: progress tracking and short-term correction

Core report order:
- chain-achievement
- channel-split
- owner-split when needed
- daily-trend when change evidence is needed

## Weekly

- meeting cadence: weekly
- title scope: Tuesday to next Monday
- MTD scope for chain-achievement: month start to Monday
- purpose: progress + channel diagnosis + accountable actions

Core report order:
- chain-achievement
- channel-split
- owner-split
- daily-trend when fluctuations matter
- detail drill-down when a weak channel needs confirmation

## Monthly

- meeting cadence: monthly
- scope: full month
- purpose: structural review, budget efficiency, next-month optimization

Core report order:
- chain-achievement
- channel-split
- owner-split
- daily-trend
- detail drill-down

## Shared Writing Rules

- Findings before summary
- Result metrics before process metrics
- Process metrics before action items
- Every action should point to a named channel and a named metric gap
- Start from the selected business scope, not from a hardcoded default scope
- Let the discovered issue decide which supporting reports and tables to use
